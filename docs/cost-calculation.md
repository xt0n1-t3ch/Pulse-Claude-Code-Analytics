# Cost Calculation Logic

## Overview

Pulse calculates token costs using Anthropic's official per-million-token pricing. Costs are computed locally from JSONL session data — no external billing API is consulted. This means displayed costs are **estimates** based on the pricing table below.

## Token Categories

Claude Code session transcripts report four token types:

| Category | Field | Description |
|----------|-------|-------------|
| **Input** | `input_tokens` | Total input tokens sent to the API. **Includes** cache_creation and cache_read tokens. |
| **Output** | `output_tokens` | Tokens generated by the model. |
| **Cache Write** | `cache_creation_input_tokens` | Tokens written to the prompt cache (new cache entries). |
| **Cache Read** | `cache_read_input_tokens` | Tokens served from the prompt cache (cache hits). |

### Pure Input

Since `input_tokens` includes cache tokens, we derive **Pure Input** for accurate breakdowns:

```
pure_input = input_tokens - cache_creation_tokens - cache_read_tokens
```

This represents tokens that were neither cached nor served from cache — truly "new" input.

## Pricing Table

All prices are per 1 million tokens. The code uses the **5-minute cache write** rate since Claude Code JSONL doesn't distinguish cache TTL (5-min vs 1-hour).

| Model | Input | Output | Cache Write (5m) | Cache Write (1h) | Cache Read |
|-------|-------|--------|------------------|-------------------|------------|
| Opus 4.5 / 4.6 | $5.00 | $25.00 | $6.25 (1.25x) | $10.00 (2x) | $0.50 |
| Opus 4.0 / 3 (legacy) | $15.00 | $75.00 | $18.75 (1.25x) | $30.00 (2x) | $1.50 |
| Sonnet (all versions) | $3.00 | $15.00 | $3.75 (1.25x) | $6.00 (2x) | $0.30 |
| Haiku 4.5+ | $1.00 | $5.00 | $1.25 (1.25x) | $2.00 (2x) | $0.10 |
| Haiku 3.5 | $0.80 | $4.00 | $1.00 (1.25x) | $1.60 (2x) | $0.08 |
| Haiku 3 | $0.25 | $1.25 | $0.30 (1.25x) | $0.50 (2x) | $0.03 |

> Source: `src/cost.rs` → `model_pricing()`

## Cost Formula

For each session, cost is calculated per token type:

```
input_cost       = input_tokens × (input_per_million / 1,000,000)
output_cost      = output_tokens × (output_per_million / 1,000,000)
cache_write_cost = cache_creation_tokens × (cache_write_per_million / 1,000,000)
cache_read_cost  = cache_read_tokens × (cache_read_per_million / 1,000,000)

total_cost = input_cost + output_cost + cache_write_cost + cache_read_cost
```

## 1M Extended Context Pricing

- **GA models** (Opus 4.6+, Sonnet 4.6+): Standard pricing at any context length. No surcharge.
- **Beta models** (Opus 4.5, Sonnet 4/4.5): When total API input exceeds 200K tokens, a surcharge applies:
  - Input: 2x standard rate
  - Output: 1.5x standard rate
  - Cache: 2x standard rate
- Detection: `max_turn_api_input > 200,000` indicates extended context usage.
- Model IDs with `[1m]` suffix (e.g., `claude-opus-4-6[1m]`) are stripped before pricing lookup.

## Cache Efficiency Metrics

### Cache Hit Ratio
```
cache_hit_ratio = cache_read_tokens / (cache_read_tokens + pure_input_tokens) × 100%
```
Higher is better — means more input is served from cache.

### Cache Savings Estimate
```
savings = cache_read_tokens × input_rate - cache_read_tokens × cache_read_rate
```
This estimates how much you saved by having those tokens cached vs paying full input price.

## Output Speed (Tokens/sec)

When statusline data is available, Pulse shows real output throughput:

```
output_speed = output_tokens / (total_api_duration_ms / 1000)
```

This uses actual API call duration (not wall-clock time), so idle/reading/editing time is excluded. Only shown when `total_api_duration_ms > 0`.

## Data Sources

1. **Statusline data** (`~/.claude/discord-presence-data.json`): Most accurate for cost/duration. Provides `total_cost_usd` directly from Claude Code's own calculation, plus `total_api_duration_ms`.
2. **JSONL parsing** (`~/.claude/projects/<path>/*.jsonl`): Zero-config fallback. Parses session transcripts and calculates cost locally using the pricing table above.

When both sources are available, statusline wins for cost/model/totals while JSONL provides granular token breakdowns and activity tracking.
